## What It Is

SciFig Generate is a [Claude Code](https://docs.anthropic.com/en/docs/claude-code) skill that transforms your experimental data — CSV, TSV, Excel, or matrix files — into submission-ready scientific figures through a fully automated four-phase pipeline.

Upload your data. SciFig validates the file path, infers data structure and scientific domain, recommends chart families and statistical tests, generates Nature / Cell / Science / Lancet / NEJM / JAMA-aligned matplotlib code, runs zero-touch finalizer passes (legend contract, layout audit, heatmap label sizing, text-occlusion guards), and exports SVG / PDF figures with source data, render-QA evidence, and a methods-ready statistical report.

## Why SciFig

| Problem | Without SciFig | With SciFig |
|---|---|---|
| Journal compliance | Hand-tune rcParams per journal, re-style for every submission | One token swap (`style="nature"` → `"cell"`) re-derives 15+ parameters |
| Chart selection | Browse seaborn examples, guess what's appropriate | 121 publication-grade chart types catalogued by 13 scientific domains |
| Statistical honesty | "Just put a t-test on it" | Auto-select test by data shape; refuse inferential claims when data only support description |
| Multi-panel layout | Manual `GridSpec` per figure | 11 layout recipes mapped to narrative arcs (hero / comparison / triptych / 2×2 / …) |
| Color governance | Pick rainbow palette, ignore colorblindness | Wong / Okabe-Ito defaults, semantic mappings preserved across panels |
| Final QA | Render, eyeball, hope reviewers don't catch overlaps | Geometric overlap audit + finalizer auto-corrections + render QA agent |
| CJK labels | `Glyph N missing from font Arial` warnings, boxes instead of text | Cross-platform CJK fallback chain (YaHei / SimHei / Noto / Hiragino) filtered at runtime, zero warnings |

## Quick Start

### 1. Install the skill into Claude Code

```bash
git clone https://github.com/Techd81/SciFig.git
cp -r SciFig/.claude/skills/scifig-generate ~/.claude/skills/
```

Restart Claude Code so the skill is auto-discovered.

### 2. Trigger from a Claude Code conversation

```
> /skill scifig-generate
> FILE: /path/to/your_data.csv
> EXTRAS: I want a hero panel showing biomarker levels over time
```

Trigger keywords also work in plain prose: `generate figure`, `plot data`, `sci figure`, `科研图`, `画图`, `多 panel`.

### 3. Receive your figure bundle

```
output/
├── figures/             # SVG + PDF, vector-only, Nature dimensions
├── source_data/         # Excel-ready per-panel tables
├── render_qa/           # Layout audit, contract enforcement, geometric overlap report
├── stats_report.md      # Methods-section-ready test descriptions
├── code/                # Reproducible generator script + helpers snapshot
└── metadata.json        # Provenance, seed, journal profile, palette
```

### Finalizer — Zero-Touch Retrofit

`enforce_figure_legend_contract(...)` runs **three corrective passes** before `audit_figure_layout_contract` so common publication failures are repaired automatically — generators don't need to call these explicitly:

| Pass | What it fixes | Effect |
|---|---|---|
| `_promote_inaxes_text_safety` | Lifts every in-axes Text to `zorder≥20` and adds a rounded white bbox (alpha 0.85) | Labels buried under data curves (`ax.text` at default zorder=3) become visible |
| `_shrink_heatmap_cell_labels` | Reformats heatmap cell text via `choose_heatmap_fmt` to fit physical cell width. **Cycle-23**: integer-count fallback keeps integer labels when cells are borderline-tight, dropping only zero-value cells | `sns.heatmap(annot=True, fmt=".3f")` adapts to `.2f` / `.1f` / `.0f` without manual tuning |
| Layout audit | Reports text-vs-line / text-vs-scatter / text-vs-patch overlap > 30% | Multi-panel figures with annotation-data collisions are flagged |

**Legend contract**: exactly one shared bottom-center rounded-frame `fig.legend` per figure. In-axes legends are consolidated post-hoc. Outside-right legends forbidden by policy.

### CJK Font Fallback

The font fallback chain extends through `[Microsoft YaHei, SimHei, Noto Sans CJK SC, Noto Sans CJK JP, Hiragino Sans, DejaVu Sans]` so Chinese / Japanese / Korean axis labels render natively on Windows / Linux / macOS. The chain is **filtered at runtime** — only families actually installed on the host survive, eliminating cross-platform `findfont` warnings. `axes.unicode_minus=False` ensures CJK fonts don't break minus signs.

### Bundled-Font Discovery

Templates anchor `font.family` to commercial typefaces (Arial / Helvetica / Times New Roman) that the skill **cannot legally redistribute**. Drop legally-obtained `*.ttf` / `*.otf` / `*.ttc` files into [`assets/fonts/`](.claude/skills/scifig-generate/assets/fonts/README.md); the first `apply_journal_kernel(...)` call registers them with matplotlib `font_manager` (idempotent, exception-safe).

## Architecture

```
SciFig pipeline — 4 phases, each owns one artifact:

  user input
       │
       ▼
┌──────────────────┐    Phase 1: Data Detection
│ data file +      │ ──▶ ingest → infer structure (tidy/wide/matrix)
│ preferences      │      → detect domain → build dataProfile
└──────────────────┘
       │ dataProfile
       ▼
┌──────────────────┐    Phase 2: Chart & Stats Planning
│ chartPlan        │ ──▶ recommend chart families → plan panel blueprint
│                  │      → pick statistical tests → build palette plan
└──────────────────┘
       │ chartPlan
       ▼
┌──────────────────┐    Phase 3: Code Generation
│ styledCode       │ ──▶ apply journal kernel → generate matplotlib code
│                  │      → run finalizer (legend / layout / heatmap labels)
└──────────────────┘
       │ styledCode
       ▼
┌──────────────────┐    Phase 4: Export & Report
│ outputBundle     │ ──▶ export SVG/PDF → render-QA → source-data tables
│                  │      → stats report → requirements → metadata
└──────────────────┘
```

Each phase reads the previous phase's artifact, owns its output, and routes blocking findings back to the owning phase before completion.

## Contributing

Issues and PRs welcome. Before submitting:

1. `pytest tests/` must pass.
2. If your change affects rendering, regenerate `baseline_gen.py` outputs and verify SSIM ≥ 0.97 against committed PNGs.
3. Match the existing code patterns; don't introduce new dependencies without justification.
4. If you change a journal profile, palette, or chart contract, update both the spec file and the corresponding test.

For new chart families: generator function → registry entry → ≥ 3 corpus cases → smoke test. Follow [`specs/template-distillation-contract.md`](.claude/skills/scifig-generate/specs/template-distillation-contract.md).

## 为什么用 SciFig

| 痛点 | 不用 SciFig | 用 SciFig |
|---|---|---|
| 期刊合规 | 每次投稿手工调 rcParams | 一行 `style="nature"` 切到 `"cell"` 自动派生 15+ 参数 |
| 图表选择 | 翻 seaborn 例子，全靠猜 | 13 个领域 121 种投稿级图表已编目 |
| 统计诚实 | "随便加一个 t 检验" | 按数据形态自动选检验；数据只支持描述时拒绝推断声明 |
| 多 panel 排版 | 每图手写 GridSpec | 11 种 layout 配方对应叙事弧 |
| 配色治理 | 用彩虹色，忽视色盲 | Wong / Okabe-Ito 默认调色板；语义映射跨 panel 一致 |
| 最终质检 | 渲染、肉眼看、希望审稿人没发现重叠 | 几何重叠审计 + finalizer 自修正 + 渲染 QA agent |
| 中文标签 | `Glyph N missing from font Arial` 警告，方块替代文字 | 跨平台 CJK 回退链（YaHei / SimHei / Noto / Hiragino）运行时过滤，零警告 |

## 快速开始

### 1. 安装到 Claude Code

```bash
git clone https://github.com/Techd81/SciFig.git
cp -r SciFig/.claude/skills/scifig-generate ~/.claude/skills/
```

重启 Claude Code，技能自动被发现。

### 2. 触发

```
> /skill scifig-generate
> FILE: /path/to/your_data.csv
> EXTRAS: 我想要一个 hero panel 显示生物标志物随时间的变化
```

关键词也可以触发：`生成图表`、`画图`、`科研图`、`多 panel`、`generate figure`、`plot data`。

### 3. 导出包

```
output/
├── figures/             # SVG + PDF，纯矢量，Nature 尺寸
├── source_data/         # 每个 panel 的 Excel-ready 表格
├── render_qa/           # 布局审计 + 合约执行 + 几何重叠报告
├── stats_report.md      # 论文方法学就绪的统计描述
├── code/                # 可复现生成脚本 + helpers 快照
└── metadata.json        # 来源、随机种子、期刊配置、调色板
```

## 核心特性

- **121 种投稿级图表** 跨 13 个科研领域，每种都有专用生成函数
- **6 种期刊样式**：Nature / Cell / Science / Lancet / NEJM / JAMA，token 驱动，一行切换
- **77 案例视觉语法知识库**：rcParams kernel、zorder recipes、14 调色板、12 layout 配方、10 叙事弧
- **Finalizer 零修改自动校正**：3 pass（文本安全提升 + 热图标签适配 + 布局审计）
- **CJK 字体回退**：Windows / Linux / macOS 自动选用 YaHei / Noto / Hiragino，运行时过滤零警告
